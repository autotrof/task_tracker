## **Technical Test** 

Kami tidak mencari coding yang “kelihatan keren”, tapi kode yang rapih, masuk akal, scalable, dan menunjukkan cara berpikir engineer saat membangun aplikasi nyata. Test ini dibuat untuk melihat seberapa kuat fundamental Flutter kandidat saat bekerja dengan API, state management, dan arsitektur aplikasi. 

## **Objective** 

Buat sebuah aplikasi Flutter sederhana bernama: **“Task Tracker App”** 

Aplikasi digunakan untuk: 

- melihat daftar task 

- menambah task 

- mengubah status task 

- melihat detail task 

Backend boleh sederhana. Fokus utama ada di kualitas Flutter implementation.
